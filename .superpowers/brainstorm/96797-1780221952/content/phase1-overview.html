Phase 1: 止血 — Current vs Target Architecture

Setter Hell elimination, command routing, no-op implementations, init rollback

❌ Current: Gateway God Object + Setter Hell
Gateway (40+ fields)
├─ runtime → 30 Set*() methods
│ ├─ if r.memStore != nil
│ ├─ if r.hookMgr != nil
│ ├─ if r.dashEmitter != nil
│ └─ ... 27 more nil checks
├─ cognitiveAgent → 25 fields (duplicated)
│ ├─ own runtime copy
│ └─ same nil checks again
├─ handleInbound() → 170 lines
│ └─ 9 if-blocks for slash commands
└─ New() → 15+ sequential init*()
   └─ no rollback on failure
✅ Target: Clean DI + Router + Lifecycle
Gateway (8 subsystems only)
├─ AgentDeps (one struct, constructor injected)
│ ├─ memStore (noop, never nil)
│ ├─ hookMgr (noop, never nil)
│ ├─ emitter (discard, never nil)
│ └─ ZERO nil checks
├─ CommandRouter (map-based dispatch)
│ └─ handleInbound() → 10 lines
└─ New() → init with rollback stack
   └─ defer rollback on any failure

四项目标改动

1

AgentDeps — 统一依赖注入

一个 struct 包含所有可选依赖，用 no-op 实现替代 nil。Runtime 和 CognitiveAgent 共享同一个 deps 实例。消除 ~60 个 Set* 方法和 ~200 个 nil 检查。

2

CommandRouter — 命令路由表

map[string]CommandHandler 替代 handleInbound 里的 if-else 链。每个斜杠命令是独立函数，可单独测试。

3

No-op Implementations

noopEmitter、noopCompressor、noopInterceptor 等零实现。编译期保证接口一致，运行时零开销（空方法被内联）。

4

Init Rollback Stack

每个 init 步骤注册对应的 cleanup 函数。任一步骤失败时 defer 栈自动回滚已初始化的资源。无资源泄漏。
