# Code of Conduct

IronClaw contributors are expected to keep technical collaboration direct, respectful, and evidence-based.

## Expected Behavior

- Focus critique on code, tests, documentation, and operational impact.
- Provide concrete reproduction steps for bugs.
- Keep discussions scoped to the issue or change under review.
- State uncertainty plainly when behavior depends on local credentials, external services, Docker, or platform-specific features.
- Respect project boundaries: prototype features, future fields, and production-connected code should be labeled accurately.

## Unacceptable Behavior

- Personal attacks or harassment.
- Publishing private tokens, logs, prompts, user data, trajectories, databases, or channel identifiers without permission.
- Misrepresenting unimplemented or prototype functionality as production-ready.
- Pressuring maintainers to merge unverified changes that affect tool execution, sandboxing, permissions, or data persistence.

## Enforcement

Maintainers may edit, close, or remove issues and pull requests that do not follow this code. Repeated violations may lead to temporary or permanent loss of contribution access.
