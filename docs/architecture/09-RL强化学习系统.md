## 一、三层 RL 架构

IronClaw 使用三层强化学习系统，每层在认知循环的不同阶段发挥作用：

```
┌─────────────────────────────────────────────────────────────┐
│                    认知循环 (Cognitive Loop)                  │
│                                                              │
│  PERCEIVE ──▶ PLAN ──▶ ACT ──▶ OBSERVE ──▶ REFLECT         │
│                 │        │                     │             │
│            ┌────▼────┐ ┌▼──────────┐  ┌───────▼────────┐   │
│            │  PPO    │ │ Contextual│  │     DQN        │   │
│            │ (L2)   │ │  Bandit   │  │    (L3)        │   │
│            │         │ │  (L1)    │  │                 │   │
│            │ 计划策略│ │ 工具选择  │  │ 重规划决策     │   │
│            │ 调整    │ │          │  │                 │   │
│            └─────────┘ └──────────┘  └─────────────────┘   │
│                                                              │
│  Level 1: Contextual Bandit — 微观工具选择                   │
│  Level 2: PPO — 中观计划策略                                 │
│  Level 3: DQN — 宏观重规划决策                               │
└─────────────────────────────────────────────────────────────┘
```

## 二、状态表示（state.go）

```go
type RLState struct {
    // 任务特征
    Complexity       float64  // 0-1, 来自 PERCEIVE
    MemoryCount      int      // 相关记忆数
    ToolCount        int      // 可用工具数
    HistoryLength    int      // 会话历史长度

    // 计划特征 (PLAN 后填充)
    SubtaskCount     int
    PlanConfidence   float64
    ToolDiversity    float64  // 使用工具种类/总工具数

    // 执行特征 (ACT/OBSERVE 后填充)
    SuccessRate      float64
    FailureRate      float64
    DeniedRate       float64
    AvgToolDuration  float64

    // 反射特征
    ReflectionConfidence float64
}
```

## 三、Level 1: Contextual Bandit（工具选择）

```go
type ContextualBandit struct {
    storage    *Storage
    priorAlpha float64  // Beta 分布先验 α
    priorBeta  float64  // Beta 分布先验 β
}
```

### 算法：Thompson Sampling

```
SelectTool(state, toolNames):
    │
    ├── 对每个候选工具:
    │   ├── 查询历史: successes, failures
    │   ├── Thompson Sampling:
    │   │   θ ~ Beta(α + successes, β + failures)
    │   └── 选择 θ 最大的工具
    │
    └── 返回 ToolSelectionAction

UpdateToolSelection(state, toolName, reward):
    └── 更新该工具的 success/failure 计数
```

**作用点**：ACT 阶段，当有多个工具可选时影响选择。

## 四、Level 2: PPO（计划策略）

```go
type PPO struct {
    cfg          config.PPOConfig
    // 策略网络 (actor) + 价值网络 (critic)
    actorWeights  []float64
    criticWeights []float64
}
```

### 策略输出

```go
type PlanStrategyAction struct {
    ConfidenceAdj float64  // 对计划置信度的调整
    // 未来可扩展: 并行度调整、工具优先级等
}
```

**作用点**：PLAN 阶段，调整 TaskPlan 的整体置信度。

### PPO 更新

```
Clip Objective:
    L(θ) = min(r(θ)·A, clip(r(θ), 1-ε, 1+ε)·A)

其中:
    r(θ) = π_new(a|s) / π_old(a|s)  # 概率比
    A = 优势函数 (GAE)
    ε = clip_epsilon (默认 0.2)
```

## 五、Level 3: DQN（重规划决策）

```go
type DQN struct {
    cfg           config.DQNConfig
    qNetwork      []float64  // 当前 Q 网络
    targetNetwork []float64  // 目标 Q 网络
    epsilon       float64    // ε-greedy 探索率
}
```

### 动作空间

```go
type ReplanActionType int

const (
    ReplanActionContinue ReplanActionType = iota  // 继续，不重规划
    ReplanActionReplan                             // 重新规划
    ReplanActionAbort                              // 中止任务
)
```

**作用点**：REFLECT 阶段，当置信度低于阈值时决定是否重规划。

### DQN 更新

```
Target: y = r + γ · max_a Q_target(s', a)
Loss:   L = (Q(s,a) - y)²

目标网络软更新: 每 target_update_freq 步
ε 衰减: ε = max(ε_end, ε * ε_decay)
```

## 六、奖励系统（reward.go）

```go
Reward = w1 * TaskSuccess + w2 * Efficiency + w3 * Safety + w4 * UserSatisfaction

默认权重:
    task_success_weight:       0.5
    efficiency_weight:         0.3
    safety_weight:             0.15
    user_satisfaction_weight:  0.05
```

| 组成部分 | 计算方式 |
|----------|----------|
| TaskSuccess | 1.0 (成功) / 0.0 (失败) |
| Efficiency | 1.0 - (duration / max_duration) |
| Safety | 1.0 - (denied_count / total_tools) |
| UserSatisfaction | 基于 reflection confidence |

## 八、训练器（trainer.go）

```go
type Trainer struct {
    policy   *Policy
    cfg      config.RLConfig

}
```

### 训练循环

```
Start(ctx):
    │
    ├── 后台 goroutine
    ├── 每 update_frequency 个经验:
    │   ├── 从 buffer 采样 batch
    │   ├── PPO 更新 (if enabled)
    │   ├── DQN 更新 (if enabled)
    │   └── Bandit 统计更新
    │
    └── 每 checkpoint_frequency:
        └── 保存检查点到 SQLite
```

### 冷启动

```
cold_start_episodes: 1000

前 1000 个 episode:
    ├── 不使用 RL 策略（纯随机/默认）
    ├── 仅收集经验数据
    └── 超过阈值后开始训练和使用策略
```

## 设计亮点

1. **三层分离**：微观(Bandit)/中观(PPO)/宏观(DQN) 各司其职
2. **冷启动保护**：前 N 个 episode 不启用策略，避免早期噪声
3. **渐进集成**：RL 完全可选，不影响核心功能
4. **异步训练**：后台 goroutine 训练，不阻塞认知循环
5. **检查点恢复**：网络权重持久化到 SQLite
