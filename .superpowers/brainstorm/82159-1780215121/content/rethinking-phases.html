重新审视 · 5 阶段真的需要吗？

拆开来看，每个阶段的真实价值和问题。

每个阶段到底在做什么

PERCEIVE
→ 项目扫描 + Git 上下文 + knowledge 检索 + 内存搜索
= 上下文注入，不需要是独立阶段

PLAN
→ LLM 生成结构化任务计划
= 强制 agent 在行动前思考。
现代 Claude/GPT-4 在 tool-use 循环中自然会做这个

ACT
→ 拓扑排序 → 并行执行工具
= 真正有价值。工具编排是 5 阶段里唯一不能省略的

OBSERVE
→ 验证断言 + 归因失败
= 真正有价值。这是自愈能力的核心

REFLECT
→ 提取教训 → 决定 replan 还是完成
= 本质上是 OBSERVE 的后续决策，
不需要独立 LLM 调用

真正重要的只有两件事

1. 上下文注入（代替 PERCEIVE）
一次性完成，注入 system prompt <DYNAMIC_CONTEXT>。
不是独立阶段，是 loop 的前置步骤。

2. 自愈循环（代替 OBSERVE+REFLECT+PLAN）
ACT → 断言检查 → 失败? → 注入失败上下文 → 重试
这才是整个 cognitive agent 的核心价值。
不是三个独立阶段，是一个闭环。

PERCEIVE 和 PLAN 是 LLM 能力溢出前的产物。
当模型是 GPT-3.5 时你需要显式分阶段引导它。
当模型是 Claude 4 时它在一个 thinking block 里完成这些。

新架构 · 单一 LLM 循环 + 自愈闭环

│ CognitiveAgent.Run(task) │
│ │

│ │

│ ┌──────────────────────┐ │

│ │ → tool_calls │ │
│ │ → execute(mw) │ middleware │
│ │ → results │ │
│ │ → next turn │ │
│ └──────────────────────┘ │
│ │
│ 3. Self-Correction Hook (post-loop) │
│ ┌──────────────────────┐ │
│ │ Assertion check │ auto-verify│
│ │ → PASS → extract │ learnings │
│ │ → FAIL → inject │ + rerun │

│ └──────────────────────┘ │
│ │

│ compression, self-heal │
│ → middleware on the LLM loop │
└─────────────────────────────────────────┘
