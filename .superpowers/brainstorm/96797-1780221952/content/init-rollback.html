Phase 1 Design — Section 4: Init Rollback Stack

每个 init 步骤注册 cleanup。失败时自动回滚已初始化的资源。无泄漏。

// internal/gateway/rollback.go
type rollbackStack struct {
  cleanups []func() // LIFO order
}

func (r *rollbackStack) push(fn func()) {
  r.cleanups = append(r.cleanups, fn)
}

func (r *rollbackStack) run() {
  for i := len(r.cleanups) - 1; i >= 0; i-- {
    r.cleanups[i]()
  }
}

Gateway.New() 中的应用

func New(cfg *config.Config) (*Gateway, error) {
  gw := &Gateway{...}
  var rb rollbackStack
  defer func() { if err != nil { rb.run() } }() // ← 失败即回滚

  if err = gw.initDatabase(); err != nil { return }
  rb.push(func() { gw.db.Close() }) // ← 注册 cleanup

  if err = gw.initFeatures(); err != nil { return } // db 自动关闭

  if err = gw.initToolsAndHooks(); err != nil { return } // db + features 自动回滚

  // ... 后续 init 同理。最后一个成功 → 置 err=nil → defer 不执行
  rb.cleanups = nil // 成功：清空回滚栈，资源由 Stop() 管理
  return gw, nil
}

好处

  • 15 行基础设施，零外部依赖
  • 每个 init 调用点自动注册清理，不需要全局追踪
  • 成功路径置 nil 避免 Stop() 时重复清理

注意

  • cleanup 函数必须幂等（Close() 多次调用不应 panic）
  • 不能替代 Stop() —— 回滚只在构造失败时触发，正常停止走原有路径
