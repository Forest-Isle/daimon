轨道 2 · 测试分层策略

9 个零测试包，按风险等级分 3 层，每层不同测试深度。

测试深度标准

Level A "集成级" — 复杂子系统，需要 mock 外部依赖 + 多组件协作测试
Level B "单元级" — 独立逻辑，纯 Go 可测试，不需要外部依赖
Level C "烟雾级" — 简单/接口定义包，happy-path 编译验证即可

🔴 Level A · 集成测试（3个包）

knowledge/ (7 files + graph/ 14 files)
• store.go: SQLite FTS5 + vector 混合检索
• graph/: 实体/关系三元组 + 递归 CTE 遍历
• pipeline.go: 文档摄取流水线
• reranker.go: LLM 重排序
→ 策略: 建临时 SQLite，走完整 ingest→chunk→embed→search→rerank 链路
→ 预计: 3-4 个测试文件，~800 行

wasm/ (6 files)
• wazero 插件运行时，插件加载/执行/隔离
→ 策略: 编译最小 .wasm 测试模块，验证加载→调用→沙箱隔离
→ 预计: 2 个测试文件，~400 行

code_engine/ (3 files)
• symbol_index.go(420行): 符号索引、调用图、语义搜索
→ 策略: 用 mock 文件系统建 Go 源码样本，验证索引→查询→调用图
→ 预计: 2 个测试文件，~500 行

🟡 Level B · 单元测试（3个包）

guardian/ (1 file, guardian.go 420行)
• 质量检查流水线，规则引擎
→ 定义 mock checker，验证流水线执行和结果聚合
→ ~250 行

browser_agent/ (2 files)
• 浏览器自动化 Agent
→ mock HTTP/Playwright 接口，验证 action 编排
→ ~200 行

finetune/ (1 file)
• 微调数据集构建/导出
→ 验证 JSONL 格式输出、token 计数
→ ~150 行

🟢 Level C · 烟雾测试（3个包）

channel/ — 接口定义 + 简单工具函数
userdir/ — 用户目录路径解析
util/ — 通用工具函数（如 TruncateStr）
→ 编译验证 + 关键函数的 table-driven 测试 → 各 ~50-80 行

~10 个新测试文件 · ~2500 行测试代码
