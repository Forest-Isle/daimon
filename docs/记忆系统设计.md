## 1. 记忆系统四层架构

认知科学将人类记忆分为多种类型，Agent 记忆系统借鉴这一框架：

```
┌─────────────────────────────────────────────────────────────────┐
│                    Agent 记忆系统四层架构                         │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  Layer 1: 工作记忆（Working Memory）                      │   │
│  │  位置：LLM Context Window                                │   │
│  │  容量：有限（4K-200K tokens）                             │   │
│  │  时效：当前会话/任务                                       │   │
│  └─────────────────────────────────────────────────────────┘   │
│                              ↕ 压缩/检索                        │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  Layer 2: 情节记忆（Episodic Memory）                     │   │
│  │  位置：数据库 / 消息历史存储                                │   │
│  │  容量：较大（数千轮对话）                                   │   │
│  │  时效：会话级别，可持久化                                   │   │
│  └─────────────────────────────────────────────────────────┘   │
│                              ↕ 提炼/索引                        │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  Layer 3: 语义记忆（Semantic Memory）                     │   │
│  │  位置：向量数据库 + 知识图谱                               │   │
│  │  容量：海量（数百万文档）                                   │   │
│  │  时效：长期，缓慢更新                                      │   │
│  └─────────────────────────────────────────────────────────┘   │
│                              ↕ 调用/学习                        │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  Layer 4: 程序性记忆（Procedural Memory）                 │   │
│  │  位置：工具注册表 / 技能库 / Fine-tuned 权重              │   │
│  │  容量：有限（几十到几百个技能）                             │   │
│  │  时效：长期，通过训练更新                                  │   │
│  └─────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
```

### 1.1 工作记忆（Working Memory）

**定义：** Agent 当前活跃的思考空间，即 LLM 的 **Context Window**。

**特点：**
- **容量有限**：受模型最大 context 限制（GPT-4 128K，Claude-3.5 200K）
- **读写最快**：直接在 prompt 中访问，无需额外 IO
- **会话结束即失效**：不持久化

**包含内容：**
- System Prompt（角色定义、工具列表）
- 当前对话轮次
- 当前任务的中间状态（Thought/Action/Observation 链）
- 从其他记忆层检索到的相关信息

**工程关注点：**
- Token 计数和成本控制
- 信息优先级排序（最重要的放在开头/结尾）
- 超限时的截断策略（滑动窗口/摘要压缩）

### 1.2 情节记忆（Episodic Memory）

**定义：** 记录 Agent 与用户的**交互历史**，类比人类对特定事件的记忆。

**存储结构：**
```python
@dataclass
class Episode:
    episode_id: str
    user_id: str
    session_id: str
    timestamp: float
    messages: list[dict]      # role/content 格式的消息列表
    task_summary: str         # 该轮任务的摘要
    tools_used: list[str]     # 使用过的工具
    outcome: str              # success | failure | partial
    key_facts: list[str]      # 从该对话中提取的关键事实
    metadata: dict
```

**典型用途：**
- "上次你帮我做了什么？" → 从情节记忆检索历史
- 识别用户偏好和习惯
- 避免在同一会话中重复询问已回答的问题

### 1.3 语义记忆（Semantic Memory）

**定义：** 关于世界的**事实性知识**，不关联特定事件，通过 RAG 接入 Agent。

**典型内容：**
- 企业内部文档、产品手册
- 领域专业知识（医疗、法律、金融）
- 用户档案（用户画像、偏好设置）
- 工具使用指南和最佳实践

**访问方式：**
- 向量相似度检索（dense retrieval）
- 全文关键词检索（BM25，sparse retrieval）
- 混合检索（hybrid retrieval）

### 1.4 程序性记忆（Procedural Memory）

**定义：** Agent 的**"技能"**，即如何执行操作的知识，对应工具和技能库。

**形式：**
- **显式**：工具注册表（Python 函数 + Schema 描述）
- **隐式**：通过 Few-shot 示例教会 LLM 特定操作模式
- **训练态**：Fine-tuned 到模型权重中的专项能力（如代码生成）

## 2. 上下文压缩策略

### 2.1 为什么需要压缩

随着对话轮次增加，工作记忆（Context Window）使用量线性增长：

```
对话轮数    Context 使用量（Token 估计）
1          ~500
5          ~2,500
20         ~10,000
100        ~50,000  ← 接近部分模型上限
```

**压缩策略的目标：**
在保留关键信息的前提下，将历史上下文压缩到可管理的大小。

### 2.2 滑动窗口（Sliding Window）

**原理：** 只保留最近 N 轮对话，丢弃更早的历史。

```python
def sliding_window_context(messages: list[dict], max_turns: int = 10) -> list[dict]:
    """
    保留 system prompt + 最近 max_turns 轮对话
    """
    system_messages = [m for m in messages if m["role"] == "system"]
    non_system = [m for m in messages if m["role"] != "system"]

    # 每轮包含 user + assistant 两条消息
    recent = non_system[-(max_turns * 2):]

    return system_messages + recent
```

**优点：** 简单，无额外 LLM 调用
**缺点：** 丢失早期重要信息（如用户在第1轮说的偏好）

### 2.3 摘要压缩（Summarization）

**原理：** 将历史对话摘要化，用摘要替代原始消息。

```python
class SummaryCompressor:
    def __init__(self, llm_client, compress_threshold: int = 3000):
        self.llm = llm_client
        self.threshold = compress_threshold  # Token 数超过此值时触发压缩

    def compress(self, messages: list[dict]) -> list[dict]:
        """
        压缩策略：
        - 保留最近 N 轮（保证 Agent 有即时上下文）
        - 将更早的历史压缩为摘要
        """
        if self._count_tokens(messages) <= self.threshold:
            return messages

        system_msgs = [m for m in messages if m["role"] == "system"]
        history = [m for m in messages if m["role"] != "system"]

        # 保留最近6条（3轮）不压缩
        KEEP_RECENT = 6
        to_compress = history[:-KEEP_RECENT]
        to_keep = history[-KEEP_RECENT:]

        if not to_compress:
            return messages

        # 生成摘要
        summary = self._summarize(to_compress)

        # 构建压缩后的消息列表
        summary_message = {
            "role": "system",
            "content": f"[对话历史摘要]\n{summary}\n[摘要结束]"
        }

        return system_msgs + [summary_message] + to_keep

    def _summarize(self, messages: list[dict]) -> str:
        """调用 LLM 生成摘要"""
        formatted = "\n".join(
            f"{m['role'].upper()}: {m['content']}" for m in messages
        )
        response = self.llm.chat.completions.create(
            model="gpt-4o-mini",
            messages=[{
                "role": "user",
                "content": (
                    f"请对以下对话历史生成简洁摘要，"
                    f"重点保留：用户需求、重要决策、关键信息。\n\n{formatted}"
                )
            }],
            max_tokens=500,
        )
        return response.choices[0].message.content

    def _count_tokens(self, messages: list[dict]) -> int:
        """估算 Token 数量（简化版）"""
        return sum(len(m.get("content", "")) // 4 for m in messages)
```

### 2.4 遗忘曲线（Forgetting Curve）

受 Ebbinghaus 遗忘曲线启发，为记忆项赋予衰减权重：

```python
import math
import time

class ForgettingCurveMemory:
    """
    基于时间衰减的记忆重要性评分
    R(t) = e^(-t/S)
    R: 记忆强度, t: 时间间隔, S: 稳定性（重要信息衰减慢）
    """

    def __init__(self):
        self.memories: list[dict] = []

    def add_memory(self, content: str, importance: float = 1.0):
        """
        importance: 1.0 = 普通信息, 2.0 = 重要, 5.0 = 关键
        """
        self.memories.append({
            "content": content,
            "timestamp": time.time(),
            "base_importance": importance,
            "access_count": 0,  # 被访问次数（复习效果）
        })

    def get_active_memories(self, max_items: int = 10) -> list[dict]:
        """获取当前最相关的记忆"""
        scored = [(m, self._compute_strength(m)) for m in self.memories]
        scored.sort(key=lambda x: x[1], reverse=True)
        return [m for m, _ in scored[:max_items]]

    def _compute_strength(self, memory: dict) -> float:
        """
        记忆强度 = 基础重要性 × 时间衰减 × 访问加成
        """
        elapsed_hours = (time.time() - memory["timestamp"]) / 3600
        stability = memory["base_importance"] * 24  # 重要信息稳定性更高

        # 时间衰减
        retention = math.exp(-elapsed_hours / stability)

        # 访问加成（被多次访问的记忆衰减更慢）
        access_bonus = 1 + 0.1 * memory["access_count"]

        return retention * access_bonus

    def mark_accessed(self, memory_idx: int):
        """标记记忆被访问，增加稳定性"""
        if 0 <= memory_idx < len(self.memories):
            self.memories[memory_idx]["access_count"] += 1
```

### 2.5 重要性评分（Importance Scoring）

```python
class ImportanceScorer:
    """
    对对话内容进行重要性评分，决定是否保留
    """

    # 高重要性关键词（规则层）
    HIGH_IMPORTANCE_PATTERNS = [
        r"记住|请记住|重要|关键|注意",
        r"我的名字是|我叫|我是",
        r"偏好|不喜欢|过敏|禁忌",
        r"截止日期|deadline|DDL",
    ]

    def score(self, message: dict) -> float:
        """返回 0.0-1.0 的重要性分数"""
        content = message.get("content", "")
        score = 0.3  # 基础分

        # 规则层：关键词匹配
        import re
        for pattern in self.HIGH_IMPORTANCE_PATTERNS:
            if re.search(pattern, content):
                score += 0.3
                break

        # 角色加权：system > user > assistant
        role_weights = {"system": 0.3, "user": 0.2, "assistant": 0.1}
        score += role_weights.get(message["role"], 0)

        # 长度启发：太短的消息通常不重要
        if len(content) < 20:
            score -= 0.1

        return min(1.0, max(0.0, score))
```

### 3.2 向量数据库存储设计

```python
from typing import Optional
import numpy as np

class LongTermMemoryStore:
    """
    基于向量数据库的长期记忆存储
    生产中使用 Pinecone / Qdrant / Weaviate / pgvector
    """

    def __init__(self, vector_db_client, embedding_model):
        self.vdb = vector_db_client
        self.embedder = embedding_model
        self.COLLECTION_NAME = "agent_memories"

    def store_memory(self, memory: dict, user_id: str) -> str:
        """
        存储记忆到向量数据库
        返回 memory_id
        """
        # 生成 embedding
        text_to_embed = self._memory_to_text(memory)
        embedding = self.embedder.encode(text_to_embed)

        memory_id = f"mem_{user_id}_{int(time.time())}"

        # 存储到向量数据库
        self.vdb.upsert(
            collection=self.COLLECTION_NAME,
            points=[{
                "id": memory_id,
                "vector": embedding.tolist(),
                "payload": {
                    "user_id": user_id,
                    "content": memory["content"],
                    "memory_type": memory.get("type", "general"),
                    "importance": memory.get("importance", 0.5),
                    "timestamp": time.time(),
                    "session_id": memory.get("session_id"),
                    "tags": memory.get("tags", []),
                }
            }]
        )

        return memory_id

    def search_memories(
        self,
        query: str,
        user_id: str,
        top_k: int = 5,
        memory_types: list[str] = None,
        recency_weight: float = 0.3,
    ) -> list[dict]:
        """
        混合检索：语义相似度 + 时序 + 重要性
        """
        query_embedding = self.embedder.encode(query)

        # 构建过滤条件
        filters = {"user_id": {"$eq": user_id}}
        if memory_types:
            filters["memory_type"] = {"$in": memory_types}

        # 向量检索
        raw_results = self.vdb.search(
            collection=self.COLLECTION_NAME,
            query_vector=query_embedding.tolist(),
            limit=top_k * 3,  # 获取更多结果用于重排序
            query_filter=filters,
        )

        # 重排序：综合语义相似度、重要性、时效性
        reranked = self._rerank(raw_results, recency_weight)

        return reranked[:top_k]

    def _rerank(self, results: list, recency_weight: float) -> list:
        """综合多因素重排序"""
        now = time.time()
        scored = []

        for r in results:
            semantic_score = r.score  # 向量相似度
            importance = r.payload.get("importance", 0.5)

            # 时效性分数（越近越高）
            age_hours = (now - r.payload.get("timestamp", now)) / 3600
            recency_score = math.exp(-age_hours / (24 * 7))  # 7天半衰期

            # 综合分数
            final_score = (
                (1 - recency_weight) * semantic_score +
                recency_weight * recency_score * importance
            )
            scored.append((final_score, r))

        scored.sort(key=lambda x: x[0], reverse=True)
        return [r for _, r in scored]

    def _memory_to_text(self, memory: dict) -> str:
        """将记忆转换为用于 Embedding 的文本"""
        parts = [memory.get("content", "")]
        if tags := memory.get("tags"):
            parts.append(f"标签: {', '.join(tags)}")
        return " ".join(parts)
```

## 4. 记忆检索策略

### 4.1 三种检索策略对比

```
┌──────────────────┬───────────────┬───────────────┬──────────────────┐
│  检索策略         │  适用场景      │  优点          │  缺点             │
├──────────────────┼───────────────┼───────────────┼──────────────────┤
│  语义相似检索     │  查找相关知识  │  理解语义，不  │  可能遗漏时序关   │
│  (Dense)         │  和历史经验    │  受关键词限制  │  键上下文         │
├──────────────────┼───────────────┼───────────────┼──────────────────┤
│  时序检索         │  需要了解近期  │  简单，保证时  │  不考虑语义相关   │
│  (Temporal)      │  发生了什么    │  效性          │  性，可能引入噪声 │
├──────────────────┼───────────────┼───────────────┼──────────────────┤
│  重要性加权检索   │  关键信息的持  │  确保重要信息  │  需要准确的重要   │
│  (Importance)    │  久记忆        │  不被遗忘      │  性评估           │
└──────────────────┴───────────────┴───────────────┴──────────────────┘
```

### 4.2 混合检索实现（Hybrid Retrieval）

```python
class HybridMemoryRetriever:
    """
    混合记忆检索器
    融合语义相似、时序相关、重要性权重
    """

    def __init__(self, vector_store, keyword_index, config: dict = None):
        self.vector_store = vector_store
        self.keyword_index = keyword_index
        self.config = config or {
            "semantic_weight": 0.5,
            "recency_weight": 0.3,
            "importance_weight": 0.2,
        }

    def retrieve(
        self,
        query: str,
        context: dict,
        top_k: int = 5,
    ) -> list[dict]:
        """
        三路融合检索，使用 Reciprocal Rank Fusion (RRF) 合并结果
        """
        # 路径1：语义检索
        semantic_results = self.vector_store.search(query, limit=top_k * 2)

        # 路径2：关键词检索（BM25）
        keyword_results = self.keyword_index.search(query, limit=top_k * 2)

        # 路径3：时序检索（最近 N 条相关记忆）
        temporal_results = self.vector_store.get_recent(
            user_id=context.get("user_id"),
            limit=top_k,
            hours=24,
        )

        # RRF 融合
        merged = self._reciprocal_rank_fusion([
            semantic_results,
            keyword_results,
            temporal_results,
        ])

        # 重要性加权
        final = self._apply_importance_weight(merged)

        return final[:top_k]

    def _reciprocal_rank_fusion(
        self,
        result_lists: list[list],
        k: int = 60
    ) -> list:
        """
        Reciprocal Rank Fusion 算法
        RRF score = sum(1 / (k + rank_i))
        融合多路检索结果，无需归一化分数
        """
        scores = {}

        for results in result_lists:
            for rank, item in enumerate(results):
                item_id = item.get("id")
                if item_id not in scores:
                    scores[item_id] = {"item": item, "rrf_score": 0}
                scores[item_id]["rrf_score"] += 1 / (k + rank + 1)

        ranked = sorted(scores.values(), key=lambda x: x["rrf_score"], reverse=True)
        return [r["item"] for r in ranked]

    def _apply_importance_weight(self, items: list) -> list:
        """在 RRF 分数基础上叠加重要性权重"""
        weight = self.config["importance_weight"]
        for item in items:
            importance = item.get("importance", 0.5)
            item["final_score"] = item.get("rrf_score", 0) * (1 + weight * importance)

        return sorted(items, key=lambda x: x["final_score"], reverse=True)
```

### 5.1 记忆冲突检测

```python
class MemoryConsistencyManager:
    """
    记忆一致性管理
    处理新信息与旧记忆冲突的情况
    """

    async def update_with_conflict_check(
        self,
        new_info: dict,
        existing_memories: list[dict],
        llm_client,
    ) -> dict:
        """
        检测新信息是否与现有记忆冲突
        如果冲突，决定是否更新
        """
        # 检索相关的现有记忆
        related = self._find_related_memories(new_info, existing_memories)

        if not related:
            # 无冲突，直接存储
            return {"action": "store", "memory": new_info}

        # 让 LLM 判断是否冲突及处理方式
        conflict_analysis = await self._analyze_conflict(
            new_info, related, llm_client
        )

        if conflict_analysis["has_conflict"]:
            if conflict_analysis["new_is_more_accurate"]:
                # 新信息更准确，更新旧记忆
                return {
                    "action": "update",
                    "memory": new_info,
                    "supersedes": [m["id"] for m in related],
                }
            else:
                # 旧信息可能更准确，标记为需要确认
                return {
                    "action": "flag_for_review",
                    "memory": new_info,
                    "conflicts_with": related,
                }

        return {"action": "store", "memory": new_info}

        self,
```

### 5.2 记忆版本管理

```python
class VersionedMemory:
    """
    带版本历史的记忆存储
    支持查看记忆变更历史和回滚
    """

    def update(self, memory_id: str, new_content: str, reason: str):
        """更新记忆并保留历史版本"""
        old = self.store.get(memory_id)

        # 保存旧版本到历史
        self.history.append({
            "memory_id": memory_id,
            "version": old["version"],
            "content": old["content"],
            "timestamp": time.time(),
            "superseded_by_reason": reason,
        })

        # 更新当前版本
        self.store.update(memory_id, {
            "content": new_content,
            "version": old["version"] + 1,
            "last_updated": time.time(),
            "update_reason": reason,
        })
```

## 6. MemGPT 架构深度解析

### 6.1 核心创新：分页记忆管理

MemGPT（2023）将操作系统的**虚拟内存和分页**概念引入 Agent 记忆管理：

```
类比关系：
  操作系统                    MemGPT
  ─────────────────────────────────────
  物理内存（RAM）      ←→     Context Window（工作记忆）
  硬盘存储             ←→     外部记忆存储（向量DB/DB）
  页表（Page Table）   ←→     记忆索引（Memory Index）
  页面置换（Swapping） ←→     记忆 Paging（换入换出）
  缺页中断             ←→     记忆检索触发
```

### 6.3 MemGPT 的关键机制

**自我指导的记忆管理（Self-Directed Memory Management）：**
- LLM 可以自主决定何时将信息写入外部存储（而非依赖固定规则）
- LLM 可以主动搜索外部存储获取遗忘的信息（类比"回忆"）
- 上下文将满时，LLM 决定哪些内容可以安全"换出"到外部存储

**心跳机制（Heartbeat）：**
- 即使没有用户消息，Agent 也会定期"醒来"
- 可以主动整理记忆（压缩、归档、清理过期信息）
- 类比操作系统的后台内存管理进程

### 6.4 MemGPT 的局限与工程实践

**局限：**
- 记忆管理函数本身消耗 Context Token（overhead 较大）
- LLM 决定何时使用记忆函数，可靠性不如规则触发
- 延迟增加（每次记忆操作需要额外的 LLM 推理）

**工程化 MemGPT 思路：**
```python
class ProductionMemGPT:
    """
    生产化的 MemGPT 实现
    将自动触发（规则）和 LLM 主动触发结合
    """

    CONTEXT_PRESSURE_THRESHOLD = 0.8  # Context 使用超过 80% 时触发

    def before_llm_call(self, messages: list) -> list:
        """每次 LLM 调用前，检查是否需要记忆管理"""
        usage = self._estimate_token_usage(messages)

        if usage > self.CONTEXT_PRESSURE_THRESHOLD:
            # 自动触发：将最旧的非关键消息换出
            messages = self._auto_swap_out(messages)

        return messages

    def after_llm_response(self, response: str, messages: list):
        """LLM 响应后，提取并存储关键信息"""
        # 规则触发：检测到关键信息模式时自动存储
        key_facts = self._extract_key_facts(response)
        for fact in key_facts:
            self.archival_memory.insert(fact)
```

## 8. 代码示例：带长期记忆的对话 Agent

```python
"""
完整的带长期记忆的对话 Agent 实现
涵盖：向量存储、摘要压缩、混合检索
"""

import json
import time
from typing import Optional
from openai import OpenAI
from dataclasses import dataclass, field

# ─── 简化版向量存储（生产使用 Qdrant/Pinecone）─────────────────────────

@dataclass
class MemoryItem:
    id: str
    content: str
    embedding: list[float]
    timestamp: float
    importance: float = 0.5
    tags: list[str] = field(default_factory=list)

class SimpleVectorStore:
    """轻量向量存储（仅用于示例，生产需替换）"""

    def __init__(self):
        self.items: list[MemoryItem] = []

    def add(self, item: MemoryItem):
        self.items.append(item)

    def search(self, query_embedding: list[float], top_k: int = 5) -> list[MemoryItem]:
        import numpy as np
        if not self.items:
            return []

        q = np.array(query_embedding)
        scores = []
        for item in self.items:
            v = np.array(item.embedding)
            cos_sim = np.dot(q, v) / (np.linalg.norm(q) * np.linalg.norm(v) + 1e-8)
            scores.append((cos_sim, item))

        scores.sort(key=lambda x: x[0], reverse=True)
        return [item for _, item in scores[:top_k]]


# ─── 记忆 Agent 核心类 ─────────────────────────────────────────────────

class MemoryAgent:
    """
    带长期记忆和摘要压缩的对话 Agent
    """

    def __init__(
        self,
        user_id: str,
        model: str = "gpt-4o-mini",
        max_context_tokens: int = 3000,
        memory_top_k: int = 3,
    ):
        self.user_id = user_id
        self.model = model
        self.max_context_tokens = max_context_tokens
        self.memory_top_k = memory_top_k

        self.client = OpenAI()
        self.vector_store = SimpleVectorStore()

        # 工作记忆（当前对话历史）
        self.conversation_history: list[dict] = []

        # 压缩后的历史摘要
        self.history_summary: Optional[str] = None

    # ── 对话主入口 ──────────────────────────────────────────────────────

    def chat(self, user_message: str) -> str:
        """处理一轮对话"""

        # Step 1: 从长期记忆检索相关信息
        relevant_memories = self._retrieve_memories(user_message)

        # Step 2: 构建消息列表（含记忆上下文）
        messages = self._build_messages(user_message, relevant_memories)

        # Step 3: 调用 LLM
        response = self.client.chat.completions.create(
            model=self.model,
            messages=messages,
            temperature=0.7,
        )
        assistant_reply = response.choices[0].message.content

        # Step 4: 更新对话历史
        self.conversation_history.append({"role": "user", "content": user_message})
        self.conversation_history.append({"role": "assistant", "content": assistant_reply})

        # Step 5: 异步（或延迟）存储重要信息到长期记忆
        self._store_to_long_term_memory(user_message, assistant_reply)

        # Step 6: 检查是否需要压缩历史
        self._maybe_compress_history()

        return assistant_reply

    # ── 记忆检索 ────────────────────────────────────────────────────────

    def _retrieve_memories(self, query: str) -> list[str]:
        """从长期记忆检索相关内容"""
        query_embedding = self._get_embedding(query)
        memories = self.vector_store.search(query_embedding, top_k=self.memory_top_k)

        if not memories:
            return []

        return [m.content for m in memories]

    # ── 消息构建 ────────────────────────────────────────────────────────

    def _build_messages(self, user_message: str, memories: list[str]) -> list[dict]:
        """构建完整的消息列表"""
        messages = []

        # System prompt
        system_content = "你是一个有记忆能力的智能助手，能够记住用户的信息和偏好。"

        # 注入长期记忆
        if memories:
            memory_block = "\n".join(f"- {m}" for m in memories)
            system_content += f"\n\n【从长期记忆中检索到的相关信息】\n{memory_block}"

        messages.append({"role": "system", "content": system_content})

        # 历史摘要（如果有）
        if self.history_summary:
            messages.append({
                "role": "system",
                "content": f"【之前对话的摘要】\n{self.history_summary}"
            })

        # 当前会话的对话历史（近期）
        messages.extend(self.conversation_history[-10:])  # 最多10条

        # 当前用户消息
        messages.append({"role": "user", "content": user_message})

        return messages

    # ── 长期记忆存储 ────────────────────────────────────────────────────

    def _store_to_long_term_memory(self, user_msg: str, assistant_msg: str):
        """
        判断是否有值得长期记忆的信息
        实际生产中这里应该是异步的，不阻塞对话
        """
        # 使用 LLM 判断是否有关键信息
        extraction_response = self.client.chat.completions.create(
            model="gpt-4o-mini",  # 用小模型降低成本
            messages=[{
                "role": "user",
                "content": (
                    f"用户说：{user_msg}\n助手回复：{assistant_msg}\n\n"
                    f"是否包含需要长期记住的关键信息（用户偏好、个人信息、重要决策等）？\n"
                    f"如果是，提取关键信息（一句话）；如果否，输出 'SKIP'。"
                )
            }],
            max_tokens=100,
            temperature=0,
        )

        extracted = extraction_response.choices[0].message.content.strip()

        if extracted.upper() != "SKIP" and len(extracted) > 10:
            embedding = self._get_embedding(extracted)
            memory = MemoryItem(
                id=f"mem_{int(time.time())}",
                content=extracted,
                embedding=embedding,
                timestamp=time.time(),
                importance=0.7,
            )
            self.vector_store.add(memory)
            print(f"[记忆存储] {extracted}")

    # ── 历史压缩 ────────────────────────────────────────────────────────

    def _maybe_compress_history(self):
        """历史超过阈值时进行摘要压缩"""
        total_chars = sum(len(m["content"]) for m in self.conversation_history)
        estimated_tokens = total_chars // 4

        if estimated_tokens < self.max_context_tokens:
            return  # 不需要压缩

        # 压缩：将历史的前半部分摘要化
        mid = len(self.conversation_history) // 2
        to_compress = self.conversation_history[:mid]
        self.conversation_history = self.conversation_history[mid:]

        # 生成摘要
        formatted = "\n".join(
            f"{m['role']}: {m['content']}" for m in to_compress
        )
        compress_response = self.client.chat.completions.create(
            model="gpt-4o-mini",
            messages=[{
                "role": "user",
                "content": (
                    f"请将以下对话历史压缩为100字以内的摘要，"
                    f"保留关键信息和上下文：\n\n{formatted}"
                )
            }],
            max_tokens=200,
        )
        new_summary = compress_response.choices[0].message.content

        # 合并新旧摘要
        if self.history_summary:
            self.history_summary = f"{self.history_summary}\n[更新] {new_summary}"
        else:
            self.history_summary = new_summary

        print(f"[历史压缩] 已压缩 {len(to_compress)} 条消息")

    # ── 工具函数 ────────────────────────────────────────────────────────

    def _get_embedding(self, text: str) -> list[float]:
        """获取文本 Embedding"""
        response = self.client.embeddings.create(
            model="text-embedding-3-small",
            input=text,
        )
        return response.data[0].embedding

    def get_memory_stats(self) -> dict:
        """获取记忆系统状态"""
        return {
            "long_term_memories": len(self.vector_store.items),
            "conversation_turns": len(self.conversation_history) // 2,
            "has_summary": self.history_summary is not None,
            "estimated_context_tokens": sum(
                len(m["content"]) for m in self.conversation_history
            ) // 4,
        }


# ─── 使用示例 ──────────────────────────────────────────────────────────

if __name__ == "__main__":
    agent = MemoryAgent(user_id="user_123", model="gpt-4o-mini")

    # 模拟多轮对话
    conversations = [
        "你好，我叫张伟，是一名软件工程师，主要做 Python 后端开发。",
        "我最近在学习 Agent 开发，对 LangChain 比较熟悉。",
        "我喜欢简洁的代码风格，不喜欢过度抽象。",
        "帮我推荐一些 Agent 框架的资料。",  # 此时 Agent 应该记住用户背景
        "我叫什么名字？我的职业是什么？",  # 测试长期记忆召回
    ]

    for user_input in conversations:
        print(f"\n用户: {user_input}")
        reply = agent.chat(user_input)
        print(f"助手: {reply}")

    print("\n记忆系统状态:", agent.get_memory_stats())
```

## 9. 面试高频问题

### ❓ 面试题 1：请解释 Agent 记忆系统的四层架构，并说明它们之间如何协作？

**💡 参考答案**

四层记忆架构对应不同的存储位置和时效性：

- **工作记忆**（Context Window）：最快速，但容量最小，是 Agent 的即时思考空间。每次 LLM 调用都在这里进行。
- **情节记忆**（Conversation History DB）：记录具体交互事件，按用户+时间索引。跨会话持久化，用于"记住上次做了什么"。
- **语义记忆**（向量数据库）：非事件性的知识，通过 RAG 在需要时检索。存储产品知识、用户偏好、领域知识等。
- **程序性记忆**（工具注册表）：Agent 的能力集合，定义了 Agent 能做什么。

**协作流程：**
1. 用户输入 → 触发从情节记忆和语义记忆检索相关内容
2. 检索到的内容注入工作记忆（Context Window）
3. Agent 决策时调用程序性记忆中的工具
4. 对话结束后，关键信息从工作记忆提炼，存入情节/语义记忆

---

### ❓ 面试题 2：Context Window 满了怎么办？有哪些应对策略？各有什么权衡？

**💡 参考答案**

**策略对比：**

**1. 滑动窗口（最简单）**
- 只保留最近 N 轮对话
- 权衡：实现简单，但早期关键信息（如用户在第1轮提的需求）会丢失

**2. 摘要压缩（最常用）**
- 将旧对话用 LLM 摘要化，保留摘要
- 权衡：增加 LLM 调用次数和成本，摘要可能丢失细节，但关键信息保留率高

**3. 重要性过滤**
- 对每条消息评分，只保留重要性高的
- 权衡：需要可靠的重要性评估，但 Token 效率最高

**4. 外部检索（RAG 式）**
- 将历史存入向量库，需要时检索
- 权衡：检索有延迟和精度损失，但几乎无容量限制

**生产最佳实践：** 组合使用：始终保留最近3-5轮（保证即时上下文），对更早的历史做摘要，同时将关键事实存入向量库以备检索。

---

### ❓ 面试题 3：如何设计记忆的重要性评分系统？

**💡 参考答案**

重要性评分需要结合多个信号：

**规则信号：**
- 包含特定关键词（"记住"、"重要"、"名字"）→ 高重要性
- 消息长度（过短的消息通常是确认，不重要）
- 消息来源（System > User > Assistant）
- 是否包含数字/日期/名称（具体信息通常更重要）

**LLM 评分（高准确率但有成本）：**
```python
# 用 gpt-4o-mini 对消息打分
"以下消息包含多少值得长期记忆的信息？打分 0-10：{message}"
```

**时效性衰减：**
- 时间越久，重要性自动衰减（Ebbinghaus 遗忘曲线）
- 但被多次引用的信息抵抗衰减（"记忆强化"）

**反馈信号：**
- 用户纠正 Agent 记错信息 → 该信息重要但需要更新
- 某条记忆被频繁检索 → 提升其重要性权重

**工程实践：** 先用规则快速筛选，对中等重要性的内容再用 LLM 精确评分，而不是对所有内容都调用 LLM。

### ❓ 面试题 4：RAG 和长期记忆有什么区别？在 Agent 中如何配合使用？

**💡 参考答案**

**RAG vs 长期记忆的本质区别：**

| 维度 | RAG（知识库检索） | 长期记忆 |
|------|--------------|---------|
| 内容类型 | 通用知识、文档 | 用户特定信息和历史 |
| 更新频率 | 低（文档级更新） | 高（每次对话都可能更新） |
| 存储主体 | 共享知识库 | 用户独立空间 |
| 隐私要求 | 通常公开 | 用户私有，需要隔离 |
| 检索目的 | 补充模型知识 | 个性化和连续性 |

**在 Agent 中的配合策略：**

```
用户查询
    │
    ├──→ RAG 检索（产品知识/领域知识）→ 外部知识增强
    │
    └──→ 长期记忆检索（用户偏好/历史）→ 个性化增强
                        │
                        ↓
                    两者融合注入 Context Window
```

**优先级设计：**
- 用户的个人记忆（长期记忆）比通用知识（RAG）有更高优先级
- 当两者冲突时，以用户最新表达的信息为准

### ❓ 面试题 5：如何解决记忆检索的"召回 vs 精确"权衡？

**💡 参考答案**

记忆检索的核心张力：
- **高召回**：获取所有可能相关的记忆 → 引入噪声，浪费 Context
- **高精确**：只获取最相关的记忆 → 可能遗漏重要信息

**实践解决方案：**

**多路检索 + 重排序（最推荐）：**
1. 多路检索：语义向量检索（高召回）+ 关键词检索（高精确）
2. Reciprocal Rank Fusion 融合多路结果
3. Cross-Encoder 重排序（用更强的模型精排 top-20 → top-5）

**查询扩展（Query Expansion）：**
- 将用户查询扩展为多个等价问法，增加召回率
- 例："我的电话号码是多少" → 同时检索"手机/联系方式/电话"

**记忆分层检索：**
- 先检索高重要性记忆（精确优先）
- 如果相关性不足，再扩展到低重要性记忆（召回补充）

**LLM-as-Reranker：**
- 对检索到的 top-K 记忆，让 LLM 判断与当前查询的相关性
- 成本较高，适合关键场景（如医疗助手的记忆检索）

### ❓ 面试题 6：MemGPT 和普通的对话历史管理有什么根本区别？

**💡 参考答案**

**根本区别在于主动性：**

普通对话历史管理：
- 被动的：由工程师编写规则决定保留哪些历史
- 固定的：压缩策略在代码中硬编码
- 无感知的：LLM 不知道上下文管理的存在

MemGPT：
- 主动的：LLM 自己决定何时读写外部记忆
- 动态的：LLM 根据当前任务需求灵活调用记忆管理函数
- 有感知的：LLM 知道自己的"记忆有限"，并主动管理

**类比：**
- 普通管理 = 操作系统自动换页（内核态）
- MemGPT = 程序员手动管理内存（用户态，如早期 C 语言）

**工程权衡：**
MemGPT 的主动管理使记忆行为更灵活，但也引入了不确定性（LLM 可能在不该存的时候存，或忘记存重要信息）。生产中通常采用折中方案：规则自动触发 + LLM 可选主动管理。

### ❓ 面试题 7：如何保证用户的记忆数据安全和隐私？

**💡 参考答案**

**隔离层面：**
- **数据隔离**：每个用户的记忆存储在独立的命名空间（向量 DB 的 collection 或 namespace），严禁跨用户查询
- **检索隔离**：检索时必须带上 `user_id` 过滤条件，在数据库层面强制隔离（而非应用层）

**加密层面：**
- 记忆存储时对敏感字段加密（用户 ID、联系方式、财务信息）
- Embedding 本身也可能泄露语义信息，考虑 Embedding 混淆技术

**访问控制：**
- 记忆访问需要用户认证（Bearer Token）
- 用户可以查看、删除自己的记忆（GDPR 的"被遗忘权"）
- 管理员访问记忆需要审计日志

**合规层面：**
- GDPR：支持用户请求删除所有记忆（Right to Erasure）
- 数据保留策略：超过 N 天未使用的记忆自动清理
- 明确告知用户哪些信息会被记忆（用户知情同意）

### ❓ 面试题 8：如何在多用户并发场景下保证记忆的读写一致性？

**💡 参考答案**

**并发场景的典型问题：**
- 同一用户的两个 Agent 实例同时运行，同时更新用户偏好 → 写写冲突
- 一个 Agent 在读取记忆的同时，另一个 Agent 在更新该记忆 → 读写一致性

**解决策略：**

**乐观锁（Optimistic Locking）：**
```python
# 更新时带版本号，版本不匹配则更新失败
def update_memory(memory_id, new_content, expected_version):
    current = store.get(memory_id)
    if current.version != expected_version:
        raise ConflictError("记忆已被其他实例修改")
    store.update(memory_id, new_content, version=expected_version + 1)
```

**用户级别的分布式锁：**
- 对同一用户的写操作加 Redis 分布式锁
- 锁粒度：用户 ID（不是全局锁）
- 锁超时：避免死锁（10-30秒）

**最终一致性（Eventual Consistency）：**
- 对于读操作，接受短暂的旧数据（读已提交的快照）
- 写操作异步同步（写入队列，后台处理）
- 适合对实时一致性要求不高的记忆（如用户偏好）

**实践中**：大多数 Agent 系统中同一用户的并发实例较少，用户级别的分布式锁已经足够，不需要引入复杂的 MVCC 机制。

### ❓ 面试题 9：如何评估记忆系统的质量？

**💡 参考答案**

**检索质量指标：**
- **召回率（Recall@K）**：K 个检索结果中包含相关记忆的比例
- **精确率（Precision@K）**：K 个检索结果中相关记忆的比例
- **MRR（Mean Reciprocal Rank）**：相关记忆在排序中的平均倒数位置

**端到端质量指标：**
- **记忆利用率**：Agent 实际使用了多少从记忆中检索到的信息（避免检索了但无用）
- **跨会话一致性**：用户在会话 B 中问及会话 A 的信息，Agent 答对的比例
- **遗忘率**：重要信息被错误遗忘（未存储或被压缩丢失）的比例

**压缩质量指标：**
- **信息保留率**：压缩后的摘要包含多少原始关键信息（LLM 评分）
- **压缩比**：原始 Token 数 / 压缩后 Token 数
- **语义一致性**：摘要与原文的向量相似度

**评估方法：**
构建 Memory QA 测试集：在多轮对话后，提问"用户的姓名/职业/偏好是什么"，检验 Agent 能否从记忆中正确回答。这是最直接的端到端评估。

### ❓ 面试题 10：描述一个你在生产中遇到的记忆系统问题，以及如何解决的。

**💡 参考答案**（典型生产案例参考）

**问题场景：记忆污染（Memory Contamination）**

在一个客服 Agent 中，当用户更正了之前提供的错误信息（如更改地址），但旧的错误信息和新的正确信息都存储在了长期记忆中，导致 Agent 在后续对话中"记忆矛盾"，无法确定应该用哪个地址。

**根因分析：**
- 记忆系统只负责"增加"新记忆，没有处理"更新/覆盖"逻辑
- 新旧信息对同一事实有矛盾，但检索时两者都被召回

**解决方案：**
1. **冲突检测层**：每次存储新记忆前，检索语义相似的现有记忆，让 LLM 判断是否存在矛盾
2. **记忆替代机制**：如果新信息是旧信息的更正，标记旧记忆为 `superseded`，检索时过滤掉
3. **事实实体化**：对关键个人信息（姓名、地址、电话）使用结构化存储（关系型 DB），而非向量存储，更新时直接覆盖
4. **用户确认机制**：当检测到新旧信息矛盾时，主动询问用户确认（"您之前提供的地址是 X，现在是否要更新为 Y？"）

**效果：** 解决后，记忆矛盾问题下降了 90%，同时用户对 Agent"记忆准确性"的满意度提升了 35%。
