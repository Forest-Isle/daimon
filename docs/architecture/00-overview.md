# 00 · 总览：Daimon 是什么

## 一句话

Daimon 是一个**主权个人代理**——一个常驻后台的 daemon，把世界的一切变化（邮件、消息、定时、文件改动）统一成事件流，对每个事件判断"值不值得醒来"，醒来则跑一段**带强制交账的认知情节**，一切副作用经过**可逆性分级的行动层**，所有事实沉淀进**唯一事实源世界模型**，并在空闲时离线整固、自我蒸馏、回放评测。

## 命名：为什么叫 Daimon

`Daimon`（δαίμων）三重契合：

1. **daemon**：unix 常驻后台进程。这个代理字面上就是一个 daemon——永远在跳的事件心脏。
2. **daimonion**：苏格拉底的守护灵——只对一个人低语、只忠于一个委托人的内在声音。主权个人代理的本质：单一委托人、结构性忠诚、local-first。
3. **论题同构**：希腊人认为 daimon 是人格中不死的部分。本项目的中心命题正是 **agent = 不死的部分（传记/信任/技能/价值观），model = 会换的认知引擎**。

落地：二进制 `daimon`，用户目录 `~/.daimon`，Go 模块 `github.com/Forest-Isle/daimon`。仓库目录历史名仍为 `IronClaw`。

## 从 IronClaw 到 Daimon：重铸缘起

前身 **IronClaw** 是一个工程骨架扎实的 coding-agent runtime：组合根装配、工具拦截链、FTS5+向量混合记忆、分层上下文压缩。但 `ARCHITECTURE_REVIEW.md` 的评审指出它的两处硬伤：

1. **执行内核停在朴素 ReAct**——`while(tool_calls){...}`，`maxIter` 到顶即停，无显式计划/反思/验证闭环。
2. **安全与可评测性近乎为零**——零隔离宿主执行 + 子串黑名单可绕过；无法量化"改了 prompt 后变好还是变坏"。

`DAIMON_BLUEPRINT.md` 给出的不是修补，而是**重铸**：把"一个更聪明的聊天机器人"重铸为"一个跑得越久越便宜、问得越来越少、被托付得越来越多的常驻进程"。采用**绞杀者模式**（strangler）——新包在旁边长，gateway 逐步换接线，旧路径跑通新路径验收后再拆，每个阶段二进制都可用。

至今蓝图 §4 的全部 13 个模块、数据层与自治环已基本建成（见 `MEMORY.md` 终态判定）。

## 宪法：七条不变量

每个模块设计必须同时满足，冲突时按序号优先。这是理解整个系统设计取舍的钥匙。

| # | 不变量 | 含义 | 落地判据 |
|---|---|---|---|
| 1 | **状态在外** | 一切持久信息住在世界模型（磁盘 / SQLite），上下文是从状态重组出来的缓存，任何时刻丢弃上下文不丢失事实。 | 杀掉运行时内存，仅凭 `~/.daimon` 重启，代理对"我是谁/在帮你做什么/上周发生了什么"答案一致。 |
| 2 | **换脑无感** | 任何模块不得依赖特定模型的行为特征；模型 ID 只出现在 `mind` 包和配置里。 | 换模型 → 回放评测回归数为零。 |
| 3 | **交账强制** | 每个情节必须以 schema 校验的 `Outcome` 收尾；每个行动必须留下 `Receipt`。无账不行动。 | 交账率 >98%（salvaged <2%）。 |
| 4 | **可逆优先** | 行动默认按最保守类别处理；升级自治必须有信任账本的战绩支撑；不可逆高风险永远人签。 | Irreversible 类信任等级封顶 `HoldThenAuto`，永不 `FullAuto`。 |
| 5 | **认知是贵的** | 任何"每个事件都调一次大模型"的设计直接否决；成本逐级过滤（规则 → 小模型 → 大模型）。 | 路由成本占总成本 <5%。 |
| 6 | **本地主权** | 传记数据不出本机（API 调用中的瞬时上下文除外）；全部行动可审计、可回放。 | 世界模型、信任账本、技能库、价值文档全部 local-first。 |
| 7 | **不替模型思考** | 禁止新增"注入提示替模型规划/反思"类机制（两个例外见下）。模型变强时 runtime 不应变成阻力。 | `reflection.go`、plan 注入、Reflexion 已删除。 |

**第 7 条的两个框架强制例外**（不替模型思考，但强制模型交账）：

1. **退出契约**：情节必须以结构化 `Outcome` 收尾（schema 校验），否则架构失忆。这是整个范式的承重墙——见 [04-episode.md](04-episode.md)。
2. **客观验证**：当行动有客观判据（测试、编译、断言）时，框架直接跑判据，不是"提示模型去验"。

## 设计哲学：铁打的爪，流水的脑

这份架构的每个设计决定都通过同一个测试：**三年后换了三代模型，它还在不在。**

- **在的**——世界模型、信任账本、技能库、价值文档、回放数据——全部能在新模型上重投。
- **不在的**——提示注入、反思脚手架、为当前模型缺陷打的补丁——一行不写。

由此推出的具体取舍贯穿全篇：

- **状态在外**（不变量 1）→ 情节是无状态的，续跑靠从世界模型重组而非反序列化 checkpoint。
- **认知是贵的**（不变量 5）→ 注意力路由是成本结构的根基，绝大多数事件在规则层被廉价处理。
- **交账强制**（不变量 3）→ `episode_close` 是保留工具，模型不交账框架就 salvage 兜底并打标，北极星指标盯死 salvaged 率。
- **可逆优先**（不变量 4）→ 行动层按可逆性分域：代码域用 OS 沙箱，生活域用可逆性分类 + 信任账本 + hold 队列。
- **复利**（sleep）→ 情节（贵）→ 技能（便宜）→ 反射（免费）。每个转正技能把一类认知成本永久清零。

## 北极星指标

| # | 指标 | 含义 | 方向 |
|---|---|---|---|
| 1 | 提案采纳率 | 预期质量 | ↑ 目标 >50% |
| 2 | 自治率（无审批行动占比） | 信任深度 | ↑（护栏：纠正/回滚率 <2%） |
| 3 | 单位价值认知成本 | 复利在发生 | ↓ 持续 |
| 4 | "从零回答"率（已告知事项被再问） | 连续性 | → 0 |
| 5 | 换脑回归数（回放） | 身份在状态里 | → 0 |
| 6 | salvaged 交账率 | 退出契约健康度 | <2% |
| 7 | WakeUser 漏报 | 路由安全 | = 0（零容忍） |
