Phase 1 Design — Section 2: CommandRouter

把 handleInbound 从 170 行砍到 10 行，每个命令独立可测

❌ Before: if-else 链
func (gw *Gateway) handleInbound(...) {
  if msg.Text == "/tasks" {
    gw.handleTasksCommand(...)
    return
  }
  if strings.HasPrefix(msg.Text, "/team ") {
    result := gw.handleTeamCommand(...)
    ...
  }
  if msg.Text == "/mode" ||
    strings.HasPrefix(msg.Text, "/mode ") {
    ...
  }
  // ... 6 more if-blocks
  // 170 lines total
}
✅ After: Router dispatch
// gateway/router.go
type CommandHandler func(
  ctx, ch, msg,
) string // returns response

var commands = map[string]CommandHandler{
  "/tasks": gw.handleTasks,
  "/team": gw.handleTeam,
  "/mode": gw.handleMode,
  "/feature": gw.handleFeature,
  "/config": gw.handleConfig,
  "/compact": gw.handleCompact,
  "/model": gw.handleModel,
  "/new": gw.handleReset,
  "/start": gw.handleReset,
}

// handleInbound → 10 lines
func (gw *Gateway) handleInbound(...) {
  if cmd, ok := commands[
    msg.Text]; ok {
      resp := cmd(ctx, ch, msg)
      // send + return
  }
  // prefix match for /team, /mode,
  // /feature, /model with args
}

前缀匹配策略

type commandEntry struct {
  handler CommandHandler
  exact bool // true = exact match only (e.g. /new)
                   // false = prefix match (e.g. /feature enable x)
}

var commandTable = map[string]commandEntry{
  "/tasks": {gw.handleTasks, true},
  "/team": {gw.handleTeam, false},
  "/mode": {gw.handleMode, false},
  "/feature": {gw.handleFeature, false},
  "/config": {gw.handleConfig, true},
  "/compact": {gw.handleCompact, true},
  "/model": {gw.handleModel, false},
  "/new": {gw.handleReset, true},
  "/start": {gw.handleReset, true},
}

// dispatch: exact match first, then prefix scan

好处

  • 加新命令 = 改 map，不改 handleInbound
  • 每个 handler 独立测试（mock ch 接口）
  • 路由逻辑 10 行，出错概率趋零

注意

  • /team 和 /mode 共享前缀匹配——/team 需排在 /tasks 前面避免误匹配
  • Handler 签名统一后，复杂的 handleFeature 需要从 msg.Text 自己解析 args
